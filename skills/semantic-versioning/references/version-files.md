# Version File Discovery and Update

This reference documents how to find and update version strings in the version
files that different ecosystems use, so that `set-version` and `bump-version`
can keep all files consistent.

---


## Discovery Order

When a version command runs, inspect the repository root and immediate
subdirectories for these files in order. Report every file found and the
version string it contains. If the files disagree, stop and ask before proceeding.

---


## Version File Formats by Ecosystem

### Plugin / skill manifests (this repo's convention)

| File | Location | Format |
| --- | --- | --- |

| `.claude-plugin/plugin.json` | repo root | `"version": "X.Y.Z"` |
| `plugin.json` | repo root | `"version": "X.Y.Z"` |

```json
{
  "version": "1.4.0"
}
```text

Update with a targeted JSON edit; preserve all other fields.

---


### Node.js / JavaScript / TypeScript

| File | Format |
| --- | --- |

| `package.json` | `"version": "X.Y.Z"` |
| `package-lock.json` | `"version": "X.Y.Z"` (also `packages[""].version`) |

```json
{
  "version": "1.4.0"
}
```text

`package-lock.json` must be kept in sync with `package.json`. Run
`npm install --package-lock-only` after editing `package.json` to regenerate
the lock file entry, or edit both files together.

---


### Python

| File | Format |
| --- | --- |

| `pyproject.toml` | `version = "X.Y.Z"` under `[project]` or `[tool.poetry]` |
| `setup.cfg` | `version = X.Y.Z` under `[metadata]` |
| `setup.py` | `version="X.Y.Z"` kwarg |
| `src/<pkg>/__init__.py` | `__version__ = "X.Y.Z"` |
| `<pkg>/__version__.py` | `__version__ = "X.Y.Z"` |

```toml
[project]
version = "1.4.0"
```text

If `pyproject.toml` uses `dynamic = ["version"]` and reads from
`__init__.py`, update only the source file — not `pyproject.toml`.

---


### Rust

| File | Format |
| --- | --- |

| `Cargo.toml` | `version = "X.Y.Z"` under `[package]` |
| `Cargo.lock` | regenerated automatically by cargo |

```toml
[package]
version = "1.4.0"
```text

Run `cargo build` or `cargo check` after editing to regenerate `Cargo.lock`.

---


### Java / JVM

| File | Format |
| --- | --- |

| `pom.xml` (Maven) | `<version>X.Y.Z</version>` under `<project>` |
| `build.gradle` (Groovy) | `version = 'X.Y.Z'` |
| `build.gradle.kts` (Kotlin) | `version = "X.Y.Z"` |
| `gradle.properties` | `version=X.Y.Z` |

```xml
<project>
  <version>1.4.0</version>
</project>
```text

For multi-module Maven projects, the parent POM version and any
`<parent>` version blocks in submodule POMs must all agree.

---


### .NET / C #
| File | Format |
| --- | --- |

| `*.csproj` | `<Version>X.Y.Z</Version>` or `<VersionPrefix>X.Y.Z</VersionPrefix>` |
| `Directory.Build.props` | same XML element |
| `AssemblyInfo.cs` | `[assembly: AssemblyVersion("X.Y.Z")]` |

---


### Go

| File | Format |
| --- | --- |

| `go.mod` | module path only; no version field |
| `version.go` / `internal/version/version.go` | `const Version = "X.Y.Z"` |

Go modules use git tags for versioning rather than in-file declarations.
Major versions ≥ 2 require a `/vN` suffix on the module path in `go.mod`.

---


### Ruby

| File | Format |
| --- | --- |

| `lib/<gem>/version.rb` | `VERSION = "X.Y.Z"` |
| `<gem>.gemspec` | `spec.version = Gem::Version.new("X.Y.Z")` or via the constant |
| `Gemfile.lock` | regenerated by bundler |

---


### PHP

| File | Format |
| --- | --- |

| `composer.json` | `"version": "X.Y.Z"` |

---


### Generic / CI configuration

| File | Format |
| --- | --- |

| `VERSION` (plain text) | `X.Y.Z\n` |
| `version.txt` | `X.Y.Z\n` |
| `.version` | `X.Y.Z\n` |
| `Makefile` | `VERSION := X.Y.Z` or `VERSION = X.Y.Z` |
| `CMakeLists.txt` | `project(... VERSION X.Y.Z ...)` |

## Update Strategy

1. **Read first.** Extract the current version from every file found.

2. **Report conflicts.** If any two files show different versions, stop and
   ask the user which is authoritative before proceeding.
3. **Edit precisely.** Change only the version string; do not reformat, sort,
   or rewrite other parts of the file.
4. **Preserve lock files.** For `package-lock.json` and `Cargo.lock`, note
   that they must be regenerated by their respective toolchains after a manual
   edit; instruct the user to run `npm install --package-lock-only` or
   `cargo check` if the lock file is tracked.
5. **Stage together.** Add all version files and `CHANGELOG.md` to the same
   git commit so the version bump is atomic.

## Detecting the Current Version (shell helper)

```bash
# package.json
node -p "require('./package.json').version"

# pyproject.toml (requires Python 3.11+ tomllib or 'tomli')

python3 -c "import tomllib; d=tomllib.load(open('pyproject.toml','rb')); print(d['project']['version'])"

# Cargo.toml
grep '^version' Cargo.toml | head -1 | sed 's/version = "\(.*\)"/\1/'

# plugin.json
jq -r '.version' .claude-plugin/plugin.json

# Plain VERSION file
cat VERSION
```text
